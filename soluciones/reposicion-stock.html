Reposición de Stock

Implementado v1
Compras · Power BI + Power Query

Modelo de BI que responde la pregunta clave de abastecimiento: ¿qué, cuánto y cuándo reponer? Cruza la demanda real (promedios móviles) con el stock físico para sustentar la propuesta de compra.

El problema

La reposición se decidía sobre planillas Excel dispersas y criterio manual: quiebres en productos de alta rotación, sobre-stock en los de baja, y retrabajo al consolidar ventas mes a mes.

La solución

Un pipeline que calcula promedios de venta en ventanas móviles de 3, 6 y 12 meses y los presenta junto al stock físico, para comparar demanda reciente vs. tendencia y decidir con datos.

Pipeline de transformación

Decisiones técnicas destacadas

Fecha de simulación configurable

Recalcula "como si fuera" otra fecha, sin tocar la consulta.

Solo meses cerrados

Excluye el mes en curso para no sesgar los promedios.

Left join desde el maestro

Ningún producto desaparece por no tener ventas: detecta stock estancado.

Blindaje de nulos

Promedios siempre numéricos: sin errores aguas abajo.

Power Query — rasgos técnicos (extractos)

1. Fecha base parametrizable. El cálculo usa una fecha de simulación si el usuario la define; si no, la fecha actual. Y siempre corta al último mes cerrado.
// Si Reporte_Simulación tiene fecha activa, se usa esa; si no, hoy.
FechaBase  = if FechaSimulacion = null
             then Date.From(DateTime.LocalNow())
             else Date.From(FechaSimulacion),
FinPeriodo = Date.EndOfMonth(Date.AddMonths(FechaBase, -1))  // último mes cerrado
2. Promedio móvil por ventana. Mismo patrón para 3, 6 y 12 meses: se suma la ventana y se divide por su número de meses (aquí, el de 3M).
// Promedio móvil de 3 meses sobre la matriz producto × mes
"Promedio_3M", (t) => List.Sum(
    Table.SelectRows(t, each
        [MesInicio] >= Inicio3M and [MesInicio] <= FinMesCerrado
    )[CantidadMes]
) / 3
3. Matriz producto × mes desde el maestro. La clave: partir del maestro de productos (no de las ventas) para no perder códigos con stock pero sin ventas recientes.
// Productos base = maestro sin duplicados (no las ventas)
ProductosBase = Table.Distinct(Table.SelectColumns(Producto, {"Codigo"})),
// ...se cruzan con todos los meses y luego con las ventas (LeftOuter);
//    los meses sin venta quedan en 0, no se descartan.
Nota: estos son extractos ilustrativos para mostrar el enfoque. La consulta completa, las fuentes y los datos de la empresa no se publican.
